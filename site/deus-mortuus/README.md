# Deus mortuus

Совместная браузерная мини-игра для поездок в ваншоте «Мир Мёртвых Богов».

## Что уже работает

- комнаты по шестизначному коду, вход игрока по имени персонажа;
- назначение нескольких игроков на один пост через панель ГМа;
- общая авторитетная симуляция в браузере ГМа и Realtime-команды игроков;
- восстановление комнаты после перезагрузки и экспорт/импорт JSON;
- фоновая симуляция не замирает, когда ГМ переключается на другую вкладку;
- облегчённый и полный режим отдельно для каждого поста;
- машинист: тяга, тормоз, отсечка, песок, карта, качество пути, инерция и волна по сцепкам;
- инженер: топка, вода, давление, пять контуров с поворотными маховиками, накопители, изоляция, перемычка и форсажный байпас;
- страж: четыре сектора восьми турелей, сигнатуры целей, ритм стрельбы, перегрев, барьер, оковы, перегородки и отсоединение вагонов;
- панель ГМа: темп, станции, окна внимания, события, поломки, ложные приборы и параметры поезда;
- выключаемый синтезированный звук, правдивые механические анимации и горизонтальный мобильный интерфейс;
- общедоступная стилизованная книга экипажа и отдельный технический справочник ГМа с формулами;
- локальный режим для разработки без внешнего сервиса.

## Локальный запуск

Из папки `site`:

```powershell
python -m http.server 8899 --bind 127.0.0.1
```

- игрок: `http://127.0.0.1:8899/deus-mortuus/`
- ГМ: `http://127.0.0.1:8899/deus-mortuus/?gm=1`
- книга экипажа: `http://127.0.0.1:8899/deus-mortuus/manual.html`
- справочник ГМа: `http://127.0.0.1:8899/deus-mortuus/gm-reference.html`

Без Supabase локальные комнаты синхронизируются только между вкладками одного браузера. Этого достаточно для разработки и демонстрации интерфейса.

Текущая сборка подключена к облачному проекту через публичный `config.js`. Для принудительного испытания без облака добавьте к адресу `?local=1` (для ГМа: `?gm=1&local=1`). Пароль базы и служебные ключи во frontend не записываются.

Локальный сервер нужен только во время разработки. После публикации `site` на GitHub Pages или другом статическом хостинге держать свой компьютер включённым не требуется: страницы раздаёт хостинг, а общие комнаты и состояние рейса обслуживает Supabase.

## Облачная настройка

1. Создать отдельный проект Supabase для DAWN.
2. Включить **Anonymous Sign-Ins**.
3. Применить по порядку миграции из `supabase/migrations`, включая `202607160001_deus_mortuus.sql`.
4. Скопировать `config.example.js` в `config.js` и указать Project URL и publishable/anon key.
5. Опубликовать папку `site` обычным способом через GitHub Pages.

Service-role key не нужен и не должен попадать в браузер. Доступ к снимкам комнат ограничен RLS, а непосредственные игровые команды передаются через временный Realtime-канал комнаты.

## Проведение

1. ГМ открывает `?gm=1`, создаёт рейс и передаёт игрокам код.
2. Игроки входят по имени персонажа и коду.
3. ГМ назначает им посты. Несколько игроков могут получить один и тот же пост без отдельных «функций помощника».
4. ГМ запускает движение. На станциях интерфейс остаётся доступным в обслуживающем режиме.
5. Поломки отмечаются в панели, но окончательно исправляются ролевыми действиями персонажей; после этого ГМ нажимает «Исправлено».
6. Резкие команды, юз, пробуксовка, превышение скорости и движение с сошедшим вагоном накапливают риск схода. ГМ может оставить автоматические последствия включёнными или удерживать порог для ручного решения.
7. После ролевого ремонта ГМ отдельно подтверждает восстановление вагона, постановку на рельсы и ремонт конкретной сцепки — состояние рейса целиком не откатывается.

## Проверка

```powershell
cd site/deus-mortuus
npm test
```

Сквозной тест в Microsoft Edge проверяет создание рейса, подключение игрока, назначение всех трёх постов, передачу команд и мобильную вёрстку:

```powershell
npm run browser-test
```
