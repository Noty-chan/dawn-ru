# Движок Техник DAWN

Статус: первый независимый срез реализован в `apps/companion/technique-engine.js`.

## Зачем он отдельный

Стол, живой лист и вкладка «Утилиты» должны обращаться к одному источнику игровых действий. Интерфейс не должен сам трактовать текст Техники или изменять Сцену набором специальных обработчиков.

Цикл движка:

```text
персонаж + правило + выборы пользователя
→ preview без изменения Сцены
→ ошибки, предупреждения, клетки, цели и команды
→ commit одной транзакцией
→ undo транзакции
```

Канонический русский текст остаётся в `data.js`. Структурированная автоматизация курируется вручную по стабильным идентификаторам: автоматически угадывать механику по формулировке правила нельзя.

## Что умеет первая версия

- проверяет наличие нужного Уровня Техники;
- проверяет дальность, смежность, границы и занятость клетки;
- строит зоны 2×2, 3×3, 5×5 и четыре направления Линии;
- находит персонажей в области и формирует выбор целей;
- создаёт области, Газ, Местность и маркеры;
- выполняет телепортацию;
- создаёт отдельное пространство и переносит выбранных участников;
- возвращает предупреждение, когда бросок или урон ещё требуют ручного решения;
- не меняет исходную Сцену во время preview;
- применяет результат атомарно и формирует обратимую транзакцию.

Registry содержит 19 структурированных правил: три Уровня «Бомбардира», «Выжженную землю», «Лей-линии», два варианта «Бесформенной силы», Дамокла, два Призыва, Печать волны, две ловушки Охотника, Тайфун, Газ Химика, «Внутренний мир», «Огромные руки», Дюрандаль и телепортацию «Двух солнц».

`techniqueCoverage(data, learnedTechniques)` дополнительно перечисляет каждый из 321 Уровней всех 107 Техник со статусом `full`, `assist` или `manual`. Непокрытая механика не исчезает из листа: она получает канонический текст и явное ручное разрешение, которое движок проводит через ту же транзакцию и журнал.

## Контракт UI

```js
const preview = DAWN_TECHNIQUE_ENGINE.preview(scene, {
  actorId,
  ruleId,
  knownLevel,
  anchor,
  destination,
  targetIds,
  orientation,
  options,
});

if (preview.ok) {
  const { scene: nextScene, transaction } =
    DAWN_TECHNIQUE_ENGINE.commit(scene, preview);
}
```

Выбранный UX «Тактический кокпит» показывает тот же результат preview на поле и не трактует правила самостоятельно. Оболочку можно менять без переписывания механики.

## Следующий технический срез

1. Добавить в участника Сцены снимок листа и собственные ресурсы вместо зависимости от глобального текущего героя.
2. Расширять `full`/`assist`-покрытие, сохраняя обязательный `manual` fallback для ещё не структурированных правил.
3. Заменить снимок в undo на обратные операции для совместной игры.
4. Добавить базовые операции ресурсов, урона, Брони, Уклонения, Ран и Эффектов.
5. Ввести события `turn.start`, `turn.end`, `round.end`, `actor.enter`, `attack.prepare` и `attack.resolve` для Газов, ловушек, Реакций и длительных правил.
6. Подключить `availableActions(actorId)` одновременно к action tray, живому листу и «Утилитам».

Пока правило не покрыто полностью, движок обязан явно помечать ручную часть. Молчаливой «автоматизации», меняющей смысл канонического правила, быть не должно.
