## Важное состояние на момент передачи

В рабочей копии есть незакоммиченный пакет улучшений. Его нельзя удалять, откатывать или перетирать:

- исправленная механика «Ва-банк»;
- учёт Рангов «Технаря»;
- отдельный тестируемый `logic.js`;
- PWA-версионирование и проверки GitHub Pages;
- новый PWA-icon;
- выбор базовых Атрибутов кнопками с автоматической перестановкой 4/3/2/2.

Проверки проходят: синтаксис JavaScript, 286 уникальных идентификаторов правил, инварианты данных и браузерный сценарий перестановки Атрибутов. Перед началом любых изменений выполнить `git status` и сохранить пользовательскую работу.

## Продуктовая планка

DAWN RU — не просто перевод и не просто калькулятор. Проект должен состоять из двух качественных продуктов:

1. законченная русская книга, которую приятно читать и использовать за столом;
2. полноценная игровая среда: создание персонажа, живой лист, сессионный стол, инструменты Нарратора, справочник и защищённая онлайн-синхронизация.

Канон механики всегда берётся из `source/translation/`, а при сомнении сверяется с английским PDF. Нельзя придумывать автоматику, меняющую правило без явного пользовательского решения.

## Безопасность и Git

- Не переиспользовать базу данных «Паники».
- Короткий код комнаты не считается средством авторизации.
- Онлайн-данные доступны только участникам кампании через нормальную авторизацию и RLS.
- Не коммитить секреты, ключи service role, пароли или локальные `.env`.
- Не отправлять изменения на GitHub без явного согласования в соответствующей задаче.
